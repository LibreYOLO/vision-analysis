YOLOv9-T vs YOLOX-S: Benchmark Comparison on NVIDIA A100

COCO val2017 · 5,000 images PyTorch FP32 March 22, 2026

Quick Verdict

YOLOX-S wins on both accuracy and speed. With 34.6 mAP@50-95 vs 30.2, YOLOX-S delivers 14.5% higher accuracy while running nearly 2x faster (44.0 FPS vs 23.2 FPS). The tradeoff: YOLOX-S uses 4.4x more parameters (8.97M vs 2.04M). If model size is your constraint, YOLOv9-T offers remarkable efficiency per parameter. Both models are available in LibreYOLO.

Accuracy

On the COCO val2017 benchmark, YOLOX-S outperforms YOLOv9-T across all accuracy metrics. The gap is most pronounced at the stricter mAP@50-95 threshold, where YOLOX-S leads by 4.4 absolute points.

Metric YOLOv9-T YOLOX-S Difference
mAP@50-95 30.2 34.6 +4.4
mAP@50 43.0 51.8 +8.8
Precision 61.5% 42.7% -18.8%
Recall 36.4% 57.0% +20.6%

An interesting nuance: YOLOv9-T actually has higher precision (61.5% vs 42.7%), meaning when it detects something, it's more likely to be correct. But YOLOX-S finds far more objects overall (57.0% recall vs 36.4%), resulting in better mAP scores.

Speed & Latency

YOLOX-S is substantially faster on the A100, achieving nearly double the throughput. The latency distribution is also tighter, with lower variance between the median and tail latencies.

Metric YOLOv9-T YOLOX-S
FPS (mean) 23.2 44.0
Latency p50 42.3 ms 22.8 ms
Latency p95 51.1 ms 28.7 ms
Latency p99 57.8 ms 29.2 ms
YOLOv9-T 23.2 FPS
YOLOX-S 44.0 FPS

Efficiency

This is where the comparison gets nuanced. While YOLOX-S wins on raw performance, YOLOv9-T is a remarkably efficient architecture when normalized by model size.

Metric YOLOv9-T YOLOX-S
Parameters 2.04M 8.97M
GFLOPs 3.98 13.46
mAP per M params 14.8 3.9
mAP per GFLOP 7.59 2.57

YOLOv9-T delivers 3.8x more mAP per million parameters and 3.0x more mAP per GFLOP. For edge deployments where model size and compute budgets are constrained, YOLOv9-T represents a compelling choice.
