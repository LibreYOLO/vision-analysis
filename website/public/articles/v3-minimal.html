YOLOv9-T vs YOLOX-S: Benchmark Comparison on NVIDIA A100

March 22, 2026 · COCO val2017 · PyTorch FP32

Two popular lightweight YOLO models, two different design philosophies. YOLOv9-T is an ultra-compact architecture with just 2 million parameters. YOLOX-S takes a more generous approach at nearly 9 million. We benchmarked both on the same A100 GPU to find out what those extra parameters buy you.

YOLOX-S wins on accuracy and speed. It delivers 34.6 mAP@50-95 at 44 FPS, versus 30.2 mAP at 23 FPS for YOLOv9-T. But YOLOv9-T is 3.8x more efficient per parameter — a significant advantage on constrained hardware.

Accuracy

The mAP gap of 4.4 points is substantial. But there's a nuance: YOLOv9-T is significantly more precise. When it detects something, it's right 61.5% of the time. YOLOX-S casts a wider net — finding more objects (57% recall) but with more false positives.

Speed

YOLOX-S runs nearly twice as fast despite having 4.4x more parameters. This likely reflects YOLOX's more GPU-friendly architecture — wider layers with simpler operations rather than YOLOv9's more complex but parameter-efficient design.

This is where YOLOv9-T shines. It extracts nearly 4x more accuracy per million parameters. For edge deployment on a Raspberry Pi or Jetson Nano, this efficiency translates directly into feasibility.

Try the Models

Both YOLOv9-T and YOLOX-S are available in LibreYOLO. Three lines to run either one:

from libreyolo import LibreYOLO

# YOLOv9-T — 2.04M params, edge-efficient
model = LibreYOLO("yolov9-t")
results = model("image.jpg")

# YOLOX-S — 34.6 mAP, speed-optimized
model = LibreYOLO("yolox-s")
results = model("image.jpg")

Reproduce These Results

Every number in this article is independently verifiable:
