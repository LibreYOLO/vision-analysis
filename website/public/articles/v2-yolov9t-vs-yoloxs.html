YOLOv9-T vs YOLOX-S: Benchmark Comparison on NVIDIA A100

COCO val2017 · 5,000 images · PyTorch FP32 · March 2026 · Benchmarked with LibreYOLO v0.1.2

On COCO val2017, YOLOX-S achieves 34.6 mAP@50-95 at 44.0 FPS while YOLOv9-T achieves 30.2 mAP@50-95 at 23.2 FPS on NVIDIA A100. YOLOX-S is 14.5% more accurate and 1.9x faster. However, YOLOv9-T is dramatically more parameter-efficient: with only 2.04M parameters (vs 8.97M), it delivers 3.8x more mAP per million parameters. Both models were benchmarked using LibreYOLO's faithful reimplementations, with retrained weights following the original papers' training recipes. Both are available via pip install libreyolo.

Accuracy

Both models are anchor-free detectors trained on COCO train2017, making this a fair comparison. YOLOX-S leads by 4.4 mAP@50-95 points. The gap widens at mAP@50, where YOLOX-S leads by 8.8 points — suggesting it's especially better at coarse localization.

Precision vs recall tradeoff: YOLOv9-T's GELAN architecture is more conservative — when it detects, it's right 61.5% of the time. YOLOX-S's SimOTA label assignment casts a wider net (57% recall) at the cost of more false positives.

Speed & Latency

Despite having 4.4x more parameters, YOLOX-S runs 1.9x faster on the A100. YOLOX's straightforward CSPDarknet convolutions are highly optimized on GPUs, while YOLOv9's PGI gradient pathways add computational overhead that parameter count alone doesn't predict.

Tail latency: YOLOX-S has remarkably tight latency — p99 is only 28% above p50 (29.2 vs 22.8 ms). YOLOv9-T's p99 is 37% above p50 (57.8 vs 42.3 ms). For real-time SLAs, YOLOX-S is more predictable.

Efficiency

YOLOv9-T's PGI and GELAN innovations specifically target parameter efficiency — and it shows. With 3.8x more mAP per million parameters, it validates the paper's core claim of extracting maximum information from minimal parameters through programmable gradient information.

Frequently Asked Questions

Which is better, YOLOv9-T or YOLOX-S?
YOLOX-S (Megvii, 2021) wins on accuracy (34.6 mAP) and speed (44 FPS). YOLOv9-T (Academia Sinica, ECCV 2024) wins on parameter efficiency (3.8x more mAP per param) with only 2.04M parameters. Both are available in LibreYOLO. Both require NMS post-processing and were trained on COCO train2017.
What are the key architectural differences?
YOLOv9-T uses GELAN (Generalized Efficient Layer Aggregation Network) with PGI (Programmable Gradient Information) for extreme parameter efficiency. YOLOX-S uses CSPDarknet with a decoupled head and SimOTA label assignment. Both are anchor-free and require NMS.
How can I reproduce these benchmark results?
Install LibreYOLO (pip install libreyolo), then run LibreYOLO("yolov9-t").val(data="coco.yaml"). All benchmarks use LibreYOLO v0.1.2 with retrained weights following the original training recipes. Results evaluated on COCO val2017 (5,000 images).
Which model for edge deployment?
YOLOv9-T at 2.04M parameters and 3.98 GFLOPs. Its GELAN architecture was designed for efficiency. Export via LibreYOLO to ONNX, TensorRT, OpenVINO, or NCNN.
