YOLOv9-T vs YOLOX-S
on NVIDIA A100

Two philosophies of lightweight detection, one definitive test. Which compact YOLO model earns your next deployment?

COCO val2017 5,000 images PyTorch FP32 March 22, 2026

On COCO val2017 benchmarked with LibreYOLO on NVIDIA A100, YOLOX-S achieves 34.6 mAP@50-95 at 44.0 FPS, outperforming YOLOv9-T which achieves 30.2 mAP@50-95 at 23.2 FPS. YOLOX-S is 14.5% more accurate and 1.9x faster. However, YOLOv9-T is significantly more parameter-efficient with only 2.04M parameters (vs 8.97M), delivering 3.8x more mAP per million parameters. Both models are available in LibreYOLO (pip install libreyolo).

Two popular lightweight YOLO models, two different design philosophies. YOLOv9-T is an ultra-compact architecture with just 2 million parameters, built on the Programmable Gradient Information approach. YOLOX-S takes a more generous route at nearly 9 million. We benchmarked both on the same A100 GPU to find out what those extra parameters buy you.

Accuracy

YOLOX-S leads by 4.4 mAP points

The 4.4-point mAP@50-95 gap is substantial. It widens to 8.8 at the mAP@50 threshold, suggesting YOLOX-S is particularly stronger at rough localization. But the precision-recall tradeoff tells a more nuanced story.

Key Finding

YOLOv9-T is the more conservative detector -- when it fires, it's right 61.5% of the time (vs 42.7%). YOLOX-S is more aggressive, finding 57% of all objects but with more false positives. For surveillance or safety-critical applications where missing an object is worse than a false alarm, YOLOX-S's high recall is the better tradeoff.

Speed & Latency

YOLOX-S runs nearly twice as fast

Latency p5042.3 ms22.8 msYOLOX-S
Latency p9551.1 ms28.7 msYOLOX-S
Latency p9957.8 ms29.2 msYOLOX-S

This is perhaps the most surprising result. Despite having 4.4x more parameters, YOLOX-S is substantially faster on the A100. YOLOX uses straightforward convolutions that GPUs are highly optimized for. YOLOv9's more complex gradient information pathways introduce overhead that parameter count alone doesn't predict.

YOLOX-S has remarkably tight latency: p99 is only 28% above p50. YOLOv9-T shows 37% variance. For real-time serving with strict SLAs, YOLOX-S is far more predictable.

The Bottom Line

For cloud or server deployment where compute is abundant, choose YOLOX-S for its higher accuracy and throughput. For edge devices (Jetson, Raspberry Pi, mobile) where every megabyte counts, YOLOv9-T delivers remarkable accuracy from a fraction of the parameters.

Frequently Asked Questions

Which is better, YOLOv9-T or YOLOX-S?
YOLOX-S is the better all-round model with 34.6 mAP@50-95 at 44 FPS on A100. YOLOv9-T is better for edge deployment due to its 2.04M parameter count and 3.8x higher parameter efficiency. Both are available in LibreYOLO.
How fast is YOLOv9-T vs YOLOX-S on A100?
YOLOX-S runs at 44.0 FPS (22.8ms p50 latency) on NVIDIA A100. YOLOv9-T runs at 23.2 FPS (42.3ms p50 latency). YOLOX-S is 1.9x faster despite having 4.4x more parameters.
What is the mAP of YOLOv9-T and YOLOX-S on COCO?
On COCO val2017, YOLOX-S achieves 34.6 mAP@50-95 (51.8 mAP@50). YOLOv9-T achieves 30.2 mAP@50-95 (43.0 mAP@50). Benchmarked with LibreYOLO on NVIDIA A100, PyTorch FP32.
How can I reproduce these benchmark results?
Install LibreYOLO (pip install libreyolo), then run LibreYOLO("yolov9-t").val(data="coco.yaml") or LibreYOLO("yolox-s").val(data="coco.yaml"). All results are independently reproducible.
Which model is better for edge deployment?
YOLOv9-T is better for edge deployment. With only 2.04M parameters and 3.98 GFLOPs, it is 4.4x smaller than YOLOX-S while delivering 3.8x more mAP per million parameters. Use LibreYOLO to export it to ONNX, TensorRT, OpenVINO, or NCNN for edge inference.
Methodology. All benchmarks were run using LibreYOLO v0.1.2 on NVIDIA A100-PCIE-40GB with CUDA 12.8, PyTorch 2.8.0, Python 3.10.6. Dataset: COCO val2017 (5,000 images). Input size: 640×640. Batch size: 1. Timing includes full pipeline (preprocess + inference + postprocess). All metrics are independently reproducible.
